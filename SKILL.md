# 学区维稳事件舆情应对全流程

> 适用于国企房地产项目（如金融街集团）旗下多项目学区分配引发维稳风险时的应对方案生成。

## 触发条件

| 信号 | 示例 |
|------|------|
| 项目未交付未纳入划片引发业主维权，或同集团多项目间学区资源张力爆发 | 恒江雅筑业主维权→可能影响理想之地 |
| 教育局划片公示后业主投诉量激增 (50+组) | 2026.05.20 江汉区学区公示事件 |
| 业主提出"跨区划片"诉求，与政策规定冲突 | 汉兴街道业主要求对口唐家墩街道学校 |
| 小红书等社交平台出现相关负面帖子 | 交房延迟帖出现"维权吧"评论 |

### Step 0: 小红书舆情快速扫描

接手学区维稳事件后，**第一步不是写报告，而是先摸清舆论场**。在小红书搜索以下关键词：

| 关键词 | 目的 |
|--------|------|
| `项目名` | 项目本身口碑 |
| `项目名 学区` | 学区相关讨论 |
| `项目名 维权/停工/交房` | 负面信号扫描 |
| `一初学苑 学区划分` | 政策落地舆情 |

每条采集20条笔记，重点看：
- 是否有直接号召维权的帖子
- 评论区情绪烈度
- 是否有业主在评论区相互串联
- 交房/学区/质量问题分别占多大比例

**采集方法**：bb-browser site xiaohongshu/search [关键词]

**⚠️ 重要**：每次 open 会创建新标签页。采集完成后必须清理标签页（通过 CDP 关闭所有 xiaohongshu 标签页），否则浏览器页面大量堆积。

**判断标准**：
- 无直接维权帖 → 舆情可控，防控为主
- 出现维权帖但互动<50 → 预警，准备回应
- 互动50+或被房产自媒体转发 → 立即启动正式响应

## 写作规范（所有输出文档通用）

本技能所有产出文档须遵守以下规则（详见 `meeting-minutes` skill）：

| 规则 | 要求 |
|------|------|
| 连字符 | 正文禁止使用 `-`。列表用 `•`，范围用「至」（如"1至2个月"） |
| Em dash（—） | 禁止使用，改用逗号或句号 |
| AI空话 | 禁止堆砌「精准撬动」「彰显」「赋能」「多方共赢」等空话 |
| 落款 | 公司名+日期行右对齐，仿宋16pt，与正文同款字体 |
| 文件命名 | `YYYYMMDD_项目_内容描述_V版本.扩展名` |
| 结构 | 两种模式对比时各自独立成节，每节含1.分析+2.应对 |
| 背景 | 开篇须有背景段，不省略 |

### Step 1: 多源文档读取与形势研判

从桌面/源文件目录读取以下类型文档：

| 文档类型 | 来源 | 典型路径 |
|---------|------|---------|
| 紧急报告（致教育局） | 桌面或源文件 | `源文件/04周边配套/学校/提请区教育局学区资源/*紧急报告*` |
| 统一解释口径（PDF） | 桌面 | `桌面/金融街恒江雅筑学区统一解释口径（2026 年）.pdf` |
| 历史教育局函件 | 源文件 | `源文件/04周边配套/学校/提请区教育局学区资源/` |
| Notion 工作记录 | Notion | 搜索"工作记录"+最近编辑时间 |

**读取注意**：PDF 需安装 pymupdf。Windows Python 环境路径为 `C:\Program Files\Python312\python.exe`。

### Step 2: 关键矛盾识别

**分析思路参考**：运用教员思维框架——先抓主要矛盾（业主当前最急的是什么？交房还是学区？），分清不同群体分别对待（急等入住户vs学区导向户vs极端维权户），重视一线信息（从小红书/业主群获取真实声音）。不急于求成，维稳是持久战。**融入时直接体现在分析逻辑中，不贴标签、不列模块名。**

学区维稳事件通常涉及**三重矛盾叠加**：

| 矛盾层 | 典型表述 | 分析要点 |
|--------|---------|---------|
| 政策与现实 | "跨街道划片无先例" vs "业主就是要上" | 查街道归属、查学校距离、查教育局已有表态 |
| 预期与落差 | "买房时口头承诺的" vs "公示没我们" | 查营销资料是否有书面承诺；没有则属预期管理失败 |
| 内部分配 | 同一个开发商凭什么我们不行 | 查两项目的街道差异、学校是否专属配建 |
| 交房延迟x学区落空（双重违约感知） | 交房遥遥无期，学区也落空 | 交房延期与学区问题叠加时，维稳压力指数级上升。业主当前不满排序通常为：交房延期 > 学区落空。若交房问题持续发酵，集团将失去分别安抚、逐个击破的腾挪空间 |

**关键判断**：业主不满的核心是预期落差而非教育质量不足——这是设计反制叙事的前提。**交房延迟是比学区问题更易燃的引信，必须并行处理。**

### Step 3: 反制叙事设计（核心步骤）

**核心原则：用存量确定性对抗增量不确定性。**

不要解释"为什么没上成"，而是证明"分配到的反而更好"。

标准反制框架：

```
原预期学校（如：一初学苑理想校区）
  → XXXX年新办整合校区
  → 首届招生，无历史数据
  → 师资待定，办学磨合期
  → 距离远（2km+）

实际分配学校（如：一初东校区）
  → XX年+成熟老牌名校
  → 普高率XX%+，可靠数据
  → XX名骨干教师固定驻校
  → 距离近（648m，步行10分钟）
```

**输出结论**：本次调整不属于学区降配，是教育局成熟优质学位优先落地的优化行为。

### 3.5 交房延迟叠加风险（常见复合场景）

学区维稳事件常与**交房延迟**并发。业主当前不满排序通常为：**交房延期 > 学区落空**。两项问题叠加形成**双重违约感知**，使集团失去分别安抚、逐个击破的腾挪空间。

应对策略：
- 将交房延迟专项应对纳入报告（见 Step 4 报告结构第3.4节）
- 在话术手册中增加交房延迟专项话术（见 Step 5 话术手册第6节）
- 核心原则：工程进度透明化（每周实拍更新），不给新的承诺性时间点，提前准备延迟补偿预案

### Step 4: 正式报告撰写（公文格式）

使用 `meeting-minutes` skill 的生成脚本生成 .docx 报告。

**写作规范（用户明确要求）：**
- 正文中禁止使用 `-` 连字符。列表用 `•`，范围用"至"（如"1至2个月"）。表格分隔线 `|---|` 属 markdown 语法，不算文本内容。
- 禁止使用 em dash（—），改用逗号或句号。
- 禁止堆砌空话（如"精准撬动""彰显国企担当""强力支撑""多方共赢""赋能"）。直接陈述事实和判断。
- 落款（公司名+日期）必须右对齐，字体仿宋16pt，与正文同款。

**报告标准结构：**

```
一、背景与形势研判
  1.1 事件经过（时间线）
  1.2 关键矛盾分析（三重矛盾）
  1.3 风险传导路径（3条路径）

二、工作组架构
  2.1 领导小组
  2.2 职能小组（维稳对接组 / 舆情管控组 / 专项协调组）
\`\`\`
三、危机项目应对措施
  3.1 业主维稳方案（三步缓冲：书面确认→信息透明→面对面）
  3.2 业主沟通策略——价值重述叙事
  3.3 教育局对接——主攻方向判断
  3.4 交房问题专项应对（如存在交房延迟问题）
       - 工程进度透明化机制
       - 交房专项对接渠道
       - 延迟补偿预案
  3.5 维稳底线预案

四、关联项目预防性措施
  4.1 话术体系微调（"产权隔离"叙事）
  4.2 舆论防火隔离（关键词监测+标准化回应）
  4.3 已购业主预防性沟通

五、多项目协同机制
  5.1 信息隔离原则
  5.2 行动协同原则
  5.3 信息传递机制

六、舆情分级响应预案
  一级（常态）/ 二级（有限扩散）/ 三级（全面扩散）
  分级响应时效：4h/12h/24h

七、保障机制
```

### Step 5: 话术手册提取（附件）

从口径PDF原文中提取关键要素，按场景整理为附件：

| 场景 | 话术类型 | 必备要素 |
|------|---------|---------|
| 小学入学 | 三校对比说服 | 小学A(实际) vs 小学B(原预期) vs 小学C(原划片) |
| 初中入学 | 三校对比说服 | 初中A(实际) vs 初中B(原预期) vs 初中C(原划片) |
| 转学 | 政策说明 | 官方流程+企业推进 |
| 争议场景 | 危机公关 | "你们当初承诺的"/"凭什么他们可以"/"我要维权" |
| 数据速查卡 | 便捷查询 | 师资/升学率/距离/通勤三校对比 |
| **交房延迟**（常见并发场景） | 危机公关+安抚 | 见下方详细说明 |

### 5.1 交房延迟话术（6个必备场景）

当学区维稳与交房延迟并发时，话术手册须补充以下场景：

| 场景 | 业主典型问法 | 话术要点 |
|------|------------|---------|
| 交房时间质疑 | 到底什么时候交房？ | 以合同约定为准，不给新承诺 |
| 停工质疑 | 是不是停工了？ | 未停工，恢复每周进度播报 |
| 信任危机 | 售楼部放假了是不是跑路了？ | 正常轮休，有客服通道 |
| 维权威胁 | 我要去维权/曝光 | 已登记诉求，X日内书面答复 |
| 前期承诺追责 | 你们拍着胸脯说6月底交房 | 承认前期沟通偏差，聚焦合同履约 |
| 群发统一模板 | 业主群内集体质疑 | 工程进度说明+恢复播报+联系方式 |

### 5.2 交房延迟统一回复模板

```
【项目名称工程进度说明】

尊敬的业主，近期大家关注的工程进度问题，现统一说明如下：

1. 项目施工正常推进，未停工。目前XXX工程已完成，正在进行XXX。
2. 即日起恢复每周工程进度播报，每周五在业主群发布实拍。
3. 具体交付时间以合同约定及后续书面通知为准。
4. 如有疑问请联系专属管家或客服热线。

感谢您的理解与支持。
```

**对话术的要求：**
- 每个话术不超过200字，便于一线客服脱稿转述
- 用"综合办学积淀、师资层级、教研体系、特色培优、办学稳定性、通勤便捷度、对口初中升学率"七大维度作为对比框架
- 关键数据用加粗+显式数字（如"普高率**70%+**"）

## 已知坑（Pitfalls）

| 坑 | 说明 |
|----|------|
| ❌ 不可在主方向判断失误 | 优先方案（跨区划片）概率极低，不应投入过多政治资源 |
| ❌ 不可就事论事解释 | 说"为什么没分到"只会激化情绪→必须做叙事转换 |
| ❌ 不可承诺无法兑现的事 | 学区问题最终由教育局决定，承诺"一定能上"会引火烧身 |
| ❌ 不可让一线随意发挥 | 所有对外口径须经集团品牌部审核，严禁超出手册范围的承诺 |
| ❌ 不可让关联项目沉默 | 沉默会让"金融街学区不稳"的标签在观望客户心中生根 |
| ⚠️ 话术数据必须准确 | 升学率/距离/师资等对比数据必须来自官方/可查证来源 |
| ⚠️ 新校区无历史数据是双刃剑 | 可用"无数据=不确定"来压低预期，但业主也会说"你怎么知道不好" |

## 三步闭环工作流

本技能产出的所有文件按以下流程流转：

```
① 源文件/ → 读取存档数据 + 输入新资料
     ↓
② 生成成果到 输出/  + 同步更新 wiki（50字起，写结论+关键数据）
     ↓
③ 输出稳定后 → 固化归档到 源文件/
```

Wiki 在第②步末尾更新，不等到第③步。详见 `meeting-minutes` skill 的「八、文件命名规范」。`
